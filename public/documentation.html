1. General Purpose

This admin panel is the operational center for the WhatsApp bot. It helps an operator understand what is happening in every conversation, what the bot already answered, which contacts are waiting for a human reply, and how to continue the conversation manually when needed.

The page is intentionally structured in layers: high-level metrics first, filters second, conversation list third, and a modal with deeper operational tools after opening a specific conversation.

2. Top Navigation Bar

Brand Area

The left side title identifies the panel as the service center. It is purely navigational and visual, giving the operator a stable point of reference while working inside the app.

Export CSV

This action downloads a spreadsheet-friendly export of all conversations. It is useful for reporting, audits, external analysis, backups, and administrative review outside the panel.

Mode Toggle

This switch changes the visual theme of the panel. It affects readability, comfort, and operator preference without changing any conversation data.

Documentation Link

This page itself exists to document the panel in detail so a team member can understand every control and every operational meaning without guessing.

3. Main Header Area

The large page heading defines the current workspace. The small cards beneath it summarize the logical building blocks of the system, such as active conversations, human follow-up, filtering capabilities, and quick reply support.

This section is not just decorative. It frames the operator mindset before interacting with live data.

4. KPI Metric Cards

The colored statistic cards give a fast operational snapshot:

  • Conversations: total number of stored conversations currently available in the panel.
  • Unread: conversations or message volume that still requires attention.
  • Pending: items that are still open or unresolved.
  • Urgent: conversations requiring fast intervention, often due to escalation or conflict.
  • Waiting for reply: conversations where the contact wrote after the last bot/admin reply.

These cards are meant for quick prioritization before diving into the conversation list.

5. Filters Toolbar

Search by name, phone, or message

This text input searches conversation-level metadata. It allows the operator to locate a contact by visible name, WhatsApp identifier, or most recent message content.

Tag filter

This field filters conversations by internal classification tags such as human handoff, risk, online, appointment, or any other custom operational label.

Date range filters

These inputs constrain results by activity window. They are useful for reviewing only recent traffic, daily traffic, or a specific operational period.

Status filter

This dropdown narrows results by conversation status, such as pending, answered, or urgent.

Refresh button

This manually reloads panel data from the backend in case the operator wants an explicit refresh instead of waiting for the auto-polling cycle.

Trash icon / clear filters

This control resets every filter field back to its neutral state, making the full conversation list visible again without manually clearing each input one by one.

6. Conversation List

Each row in the list is a compact operational summary of one WhatsApp conversation.

  • Contact name: the display name captured from WhatsApp.
  • JID: the raw WhatsApp identifier, useful for support and debugging.
  • Status badge: the current operational state of the conversation.
  • Conflict badge: if present, this indicates whether the conversation includes mild conflict, aggression, or threat-level language.
  • Unread badge: indicates new messages not yet reviewed from the panel workflow.
  • Message count: the total number of stored messages in that thread.
  • Tag strip: labels used to classify the conversation operationally.
  • Preview: the latest message summary.
  • Last bot reply / waiting line: shows whether the contact is currently waiting for a response.
  • Action controls: allow quick status change, mark as read, or open the full details modal.

7. Conversation Modal

The modal is the detailed workspace for a single conversation. It contains two tabs and multiple information layers.

Details tab

Shows structured metadata, internal notes, management actions, escalation reason, conflict level, and the full chronological history.

Reply tab

Shows the conversation in a chat-like layout and provides the manual reply composer, quick replies, file upload, and send action.

8. Detail Cards Inside the Modal

The summary cards inside the modal explain the conversation state from multiple angles:

  • Status: current operational label.
  • Unread: unread count from the operator perspective.
  • Tags: internal categorization labels.
  • Last bot reply: latest outgoing answer from the automated or manual side.
  • Time waiting for response: indicates how long the contact has been waiting since their latest incoming message.
  • Escalation reason: the reason why the conversation was sent to human follow-up.
  • Conflict level: severity level of detected conflict, if any.

9. Management Area

This block is where the operator manages conversation metadata:

  • Status selector: updates the operational state.
  • Mark as read: clears pending unread count.
  • Save changes: persists notes, tags, and status.
  • Tags input: stores custom labels separated by commas.
  • Notes field: internal-only notes for the operator team.

10. Reply Area

This is the interactive operator reply workspace.

  • Quick replies: predefined templates to speed up common answers.
  • Attachment input: used to upload images, audio, video, and documents for WhatsApp delivery.
  • Text / caption field: can be used as a normal text reply or as the caption for a selected attachment.
  • Send reply button: sends text-only replies or media + caption depending on what is currently filled in.

11. Full History

The history section keeps the original chronology of the conversation. It is useful for quality review, support context, operational auditing, and continuity when different operators handle the same contact.

Messages may include plain text, media previews, downloadable documents, timestamps, and sender direction information.

12. Linking Screen

The root route of the application shows the WhatsApp QR linking screen. If there is no valid session, the operator scans the QR once. After successful linking, the interface redirects automatically to the admin panel. If the saved WhatsApp session remains valid on the server, that linking step does not need to be repeated.
